# Contributing to DocSense Cloud

Thank you for your interest in contributing to DocSense Cloud! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies with `npm install`
4. Create a new branch for your changes

## Development Workflow

### Setting Up Your Environment

1. Ensure you have Node.js 20 or later installed
2. Run `npm install` to install dependencies
3. Run `npm run dev` to start the development server
4. The app will be available at `http://localhost:3000`

### Making Changes

1. Create a new branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Make your changes following the code style guidelines below

3. Test your changes locally

4. Commit your changes with clear, descriptive commit messages

### Code Style

- Use TypeScript for all new code
- Follow the existing code patterns and structure
- Use CSS Modules for component styling
- Keep components small and focused
- Add types for all props and state

### Submitting Changes

1. Push your branch to your fork
2. Create a Pull Request against the `main` branch
3. Provide a clear description of your changes
4. Link any related issues

## Reporting Issues

When reporting issues, please include:

- A clear description of the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Screenshots if applicable
- Browser and OS information

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).
